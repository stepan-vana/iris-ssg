Iris SSG is an open-source project and we welcome contributions from anyone. Whether you're fixing a bug, improving the documentation, or proposing a new feature — your input is valued.

## Submitting a Pull Request

If you have a fix or improvement ready, the best way to contribute is through a pull request on GitHub.

1. Fork the [Iris SSG repository](https://github.com/stepan-vana/iris-ssg)
2. Create a new branch for your change
3. Make your changes and commit them with a clear message
4. Open a pull request and describe what you've changed and why

We review all pull requests and aim to respond promptly. There's no contribution too small — typo fixes and documentation improvements are just as welcome as new features.

## Reporting Bugs & Security Issues

If you come across a bug, a visual glitch, or a security vulnerability, please let us know.

- **General bugs** — open an issue on GitHub with a description of the problem, steps to reproduce it, and your browser/OS if relevant.
- **Security vulnerabilities** — please do not open a public issue. Instead, reach out to us directly so the issue can be addressed before any public disclosure.

The more detail you provide, the faster we can address it.

## Join the Team

We're building a small open-source community around Iris SSG and are looking for people who want to be involved on an ongoing basis — not just as one-time contributors, but as part of the team.

If you're interested in helping shape the direction of the project, reviewing pull requests, or maintaining documentation, get in touch. There's no formal process — just reach out via GitHub and we'll go from there.

## Code of Conduct

We expect all contributors to engage respectfully and constructively. This is a welcoming project and we intend to keep it that way.
